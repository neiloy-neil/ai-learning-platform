
# Student and Family CRM

This feature is responsible for managing student and family information.

## Purpose

The main purpose of this feature is to have a single source of truth for all student and family data. This includes student profiles, parent profiles, family accounts, siblings, and contact preferences.

## Main Components

*   **Student Profile:** Contains all the information about a student, including their personal details, school information, and academic history.
*   **Parent Profile:** Contains all the information about a parent, including their contact details and linked students.
*   **Family Account:** Represents a family unit and links together students and parents. It also owns invoices, payments, and other family-level data.

# Lead and Enquiry Tracking

This feature is responsible for managing new enquiries and leads.

## Purpose

The main purpose of this feature is to ensure that no enquiry is lost and to provide a structured workflow for following up with new leads.

## Main Components

*   **Enquiry Form:** A public form to capture new enquiries.
*   **Lead Profile:** Contains all the information about a lead, including their contact details, the source of the enquiry, and the parent's concerns.
*   **Lead Pipeline:** A kanban-style board to track the status of each lead, from "New" to "Enrolled".

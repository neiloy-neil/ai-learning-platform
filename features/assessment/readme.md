
# Diagnostics

This feature is responsible for managing diagnostic assessments.

## Purpose

The main purpose of this feature is to provide a structured and auditable placement process for new students.

## Main Components

*   **Diagnostic Booking:** A calendar-based system for booking diagnostic assessments.
*   **Diagnostic Assessment:** The actual assessment that the student takes.
*   **Diagnostic Result:** The result of the assessment, including a placement recommendation.

# Weekly Assessments

This feature is responsible for managing weekly assessments.

## Purpose

The main purpose of this feature is to track student progress on a weekly basis and to feed the data into the academic intelligence layer.

## Main Components

*   **Assessment Assignment:** A way for teachers to assign weekly assessments to their classes.
*   **Assessment Completion:** A UI for students to complete the assessments.
*   **Result Entry/Import:** A way to record the results of the assessments, either manually or by importing them from another system.
*   **Concept Summary:** A summary of the student's performance at the concept level.
